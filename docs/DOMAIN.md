# Домен приложения personal-profile

## Назначение

`personal-profile` — модуль персонального профиля пользователя, который можно подключать в другие Angular-проекты как самостоятельный блок. Профиль отображает данные пользователя и предоставляет единую точку навигации по связанным разделам.

## Расширяемость через плагины

Функциональность расширяется плагинами (`ProfilePlugin`). Каждый плагин:

- имеет уникальный `id`;
- регистрирует маршрут (`routePath`);
- может добавлять пункт в навигацию.

Ядро приложения отвечает за layout, маршрутизацию и загрузку плагинов. Бизнес-логика плагинов изолирована в `features/`.

## Основные сущности

| Сущность         | Описание                                              |
| ---------------- | ----------------------------------------------------- |
| `PersonalProfile` | Профиль пользователя: идентификатор, имя, плагины    |
| `ProfilePlugin`  | Подключаемый модуль с маршрутом и метаданными         |
| `NavItem`        | Пункт навигации (метка + путь)                        |
| `AppContour`     | Контур развёртывания: dev, test, lt, preprod, prod    |

## Границы ответственности

- **core** — layout, конфигурация окружения, доменные типы.
- **shared** — переиспользуемые UI-компоненты и утилиты.
- **features** — изолированные фичи и плагины (security, home и др.).
